# ROADMAP

TaskManager 的目标是保持轻量、易用，同时逐步提升稳定性、可维护性和可扩展性。

## 近期重点

### 1. 稳定核心能力
- 补强调度、日志、异常场景测试
- 修复或规避 logger / handler 重复绑定问题
- 明确导入时自动创建日志目录的行为边界

### 2. 增强任务模型
- 为任务补充 `timeout`、`retry`、`cwd`、`env` 等基础配置
- 引入更清晰的任务状态模型
- 评估是否增加统一任务上下文

### 3. 完善调度能力
- 进一步明确并发模式下的失败停止语义
- 改善最终执行结果汇总
- 评估任务依赖、分组和阶段执行能力

## 中期方向

### 4. 提升开发体验
- 补齐关键模块类型标注
- 改善 rich / 标准进度条下的运行结果展示
- 完善 README 中的串行、并发、失败处理和日志示例

### 5. 增强使用方式
- 提供简单 CLI
- 评估 YAML / JSON 配置文件驱动能力

## 长期方向

### 6. 面向发布与维护
- 明确公共 API 与内部实现边界
- 建立版本号与变更记录策略
- 整理构建、验证和发布流程

## 优先级建议

建议按以下顺序推进：

1. 测试补强
2. 日志系统打磨
3. 初始化行为梳理
4. 任务配置增强
5. 并发与结果模型完善
6. 类型标注与文档
7. CLI / 配置文件支持
